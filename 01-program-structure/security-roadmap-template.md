# Roadmap de Segurança — Template (30/60/90 + 12 meses)

Este documento é um template para construir um **roadmap de Segurança** que conecte:
- riscos e prioridades (PIRs/riscos/stack)
- objetivos (OKRs)
- iniciativas (projetos/entregas)
- dependências (TI/negócio)
- métricas (KPIs)

> Objetivo: sair do “backlog infinito” e ter um plano executável e reportável.

## 2) Diretrizes (como priorizar)

Critérios recomendados:
- redução de risco (impacto x probabilidade)
- exposição real (ativos críticos)
- obrigação/regulatório (LGPD, auditoria, contrato)
- esforço e dependências
- “quick wins” que destravam execução

> Dica: priorize poucas iniciativas bem executadas.
